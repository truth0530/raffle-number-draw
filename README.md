# 현장 번호 추첨 (Raffle Number Draw)

현장에서 종이 번호표(1~N)를 나눠준 뒤, 화면에 랜덤 번호를 뽑아 띄우는 **현장 추첨 웹앱**입니다.
파워포인트의 발표자 모드/슬라이드쇼처럼 **관리자 창 + 슬라이드쇼 창(듀얼 모니터)** 으로 운영합니다.

번호 추첨 기능은 **서버·데이터베이스가 필요 없습니다.** 같은 브라우저의 두 창을
`localStorage` + `BroadcastChannel`로 동기화하므로, 정적 호스팅(Vercel 등)에 그대로 배포됩니다.

## 화면
- `/numbers/admin` — **발표자(관리자)**. 범위·개수·속도 설정, 추첨/채우기, 번호 상태 표시.
- `/numbers/show` — **슬라이드쇼**. 주소창·헤더 없는 팝업으로 열어 프로젝터에 표출.
  (관리자에서 **[슬라이드쇼 창 열기]** → 다른 모니터로 옮기고 **클릭하면 전체화면**.)

## 사용법
1. `/numbers/admin` 에서 **번호 범위(1~400)**, **뽑기 개수(20)**, **속도(초/개)** 설정 → 적용.
2. **[슬라이드쇼 창 열기]** → 프로젝터 모니터로 이동 → 클릭해 전체화면.
3. **추첨** → 번호가 비오듯 쏟아지며, 설정 속도로 **1개씩 순차 정착**(1.0초 → 20초, 0.5초 → 10초).
   각 번호는 그 순간 실시간 추첨(미리 정해지지 않음), 중복 없음.
4. 사회자가 호명하는 동안 방송실은 번호를 빠르게 표시:
   - **받음 ✓** — 확실히 받아간 사람 체크(파랑).
   - **없음 ✗** — 안 나온 번호 → 슬라이드쇼 **우측 '없음' 박스로 이동**.
   - **채우기** → 없음 개수만큼 대체 추첨(초록 강조). 잘못 뺀 번호는 없음 박스에서 탭해 되돌리기.

## 개발
```bash
npm install
npm run dev            # http://localhost:3000/numbers/admin
npm run build          # 프로덕션 빌드
```
- 스택: Next.js 16 + React 19 + TypeScript. 번호 추첨은 순수 클라이언트.
- 배포: 정적/서버리스 호스팅에 그대로 배포 가능(GitHub 푸시 → 자동 배포).

## 참고
이 저장소에는 실험적으로 개발하던 **'유리병 물리 응모 추첨'** 기능(`/stage`, `/control`, `/api/*`)도
포함되어 있으나 **보류 상태**이며, Prisma+SQLite에 의존해 서버리스 환경(Vercel)에서는 동작하지 않습니다.
현재 사용하는 기능은 위의 **번호 추첨**입니다.
